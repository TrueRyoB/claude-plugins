# explain-like-im-5 — 未知領域を最短で腹落ちさせる

依頼者（＝誰でも）に、未知領域の概念を前提ゼロから積み上げて腹落ちさせる。
目的は「知能を下げた説明」ではなく **その領域固有の前提を持っていない状態を最短で解消する**こと。落とすのは *ドメイン固有の前提* だけで、思考の解像度は落とさない。
説明の深さは相手に **聞かず**、普段の会話の仕方から推定して校正する（下記 B）。相手は自分のマシンで自分の `~/.claude/` を読むだけなので越境は無い。

## A. 出力フォーマット（層で積む）
各層は数行。壁のようなテキストにしない。

0. **前提1行** — 「◯◯は既知として説明します（違えば言ってください）」。推定 level を訂正可能な形で先に置く。
1. **30秒版** — 一言で何か ＋ **なぜ存在するか（どの問題を解くために生まれたか）**。機能より存在理由が先。
2. **登場人物** — 主要な部品/アクター 3〜5個、各1行で「何をする役か」。
3. **どう動くか（なぜ）** — 因果を1本の線で辿る（「Aが起きる→だからB→結果C」）。
4. **たとえ** — 相手の既知に繋ぐアナロジーを **1つだけ**。そして **破綻する境界を1行**（「ただし〜の点は違う」）。
5. **落とし穴** — 詰まりやすい／直感に反する点を1〜2個。
6. **次の一歩** — もっと深掘るなら何を読む/触るか。または「これで何が説明できるようになったか」。

## B. 深さの推定（このスキルの核）— 情報源の優先順位と読み方

**原則：live を最優先し、薄い時だけディスクへ escalate。高コスト源に飛びつかない。**

1. **現ターンの会話（無償・最重要）** — 依頼者の言い回し／無償で使う専門語／質問の粒度／訂正の細かさ／「自明扱いしている前提」。ここで足りれば以降は読まない。
2. **`~/.claude/history.jsonl`（安・常用）** — 「普段の」文体の最安コーパス。`jq -c '.display' ~/.claude/history.jsonl`。現ターンが薄い（cold session／短い依頼）時にまずここ。
3. **`~/.claude/CLAUDE.md` ＋ memory**（`~/.claude/projects/<cwd>/memory/` の `MEMORY.md`・`feedback-*`・あれば `user` 型）— ドメイン/嗜好の明示シグナル。あれば読む、無ければ素通り。
4. **recent transcripts（高コスト・必要時のみ）** — `~/.claude/projects/*/*.jsonl` を mtime 上位50件に絞り、`jq -c 'select(.type=="user" and .promptSource=="typed") | .message.content | select(type=="string")'` でストリーム抽出。**全文をコンテキストに載せない**（数MB級あり）。1〜3で決まらない時だけ。

**読み取る軸**：ドメイン語の自発使用と正確さ／文体の抽象度／説明を求める vs 自明扱いする対象（＝相手の frontier）／訂正の技術的細かさ／CLI・自作スキルの習熟度。**言語は日本語主体になりがち → 英語キーワード依存の判定は誤爆するので使わない。**

**校正の使い道**：推定 level が (1) どこから説明を始めるか（既知の飛ばし幅）、(2) たとえの anchor（例：エンジニアには工学アナロジー）、(3) 未定義で使ってよい専門語の量、を決める。

## C. 規律（言いかけたら止まる）

- **推定を黙って断定しかけた** → 推定は測定でなく推測。層0の前提1行で訂正可能にし、相手の反応で **即再校正**。（これは「level を聞く」ではなく、シグナル由来の訂正可能な既定値。）
- **高コスト transcript を先に読みかけた** → live と history で足りないか先に確認。足りるなら読まない。読む時も mtime50・typed・jq stream・全文非搭載を厳守。
- **専門語を定義なしで使いかけた** → 初出で即定義。定義できない語で未知を説明しない。
- **アナロジーを2つ目足したくなった** → 足さず1つに絞り、破綻境界を書く方へ回す（複数並べたくなるのは核が絞れていないサイン）。
- **「だいたいこう」で複雑さを削りかけた** → 削るのは可。ただし **「※実際は〜」1行で残す**。黙って捨てて誤った確信を植えない。
- **signal が真に薄い** → 中庸 level を既定に。ただし *対象領域* は「未知だから頼まれている」ので、そこはやや低めの既知想定に倒す。
